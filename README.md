# BIOSUMMER бот для голосования

Бот предназначен для проведения голосования за фотографии на Био Лето.
Пайплайн состоит из подготовки фотографий, нормализации структуры папок, создания таблицы голосования в SQLite и запуска Telegram-бота.

Проект рассчитан на сценарий, в котором:
- вы один раз готовите набор фотографий для конкретного года;
- бот работает несколько дней или недель;
- пользователи голосуют через Telegram;
- результаты затем извлекаются из SQLite-базы.

---

## Основаная структура проекта

```text
C:.
│   creating_voting_db.py
│   main.py
│   rename_all_entries.py
│
└───mountables
        logger.log
        voting.db
        token.txt (нужно создать перед запуском)
        entries (папка с фотографиями, нужно создать)
```

### Назначение файлов

- `rename_all_entries.py` — переименовывает папки-категории и файлы фотографий в формат, понятный боту.
- `creating_voting_db.py` — создаёт таблицу голосования в существующей SQLite-базе.
- `main.py` — запускает Telegram-бота и обрабатывает команды `/start`, `/vote`, `/cancel`.
- `requirements.txt` — список Python-зависимостей проекта.
- `mountables/voting.db` — основная SQLite-база, в которой хранятся результаты голосования за разные годы.
- `mountables/logger.log` — файл логов бота.
- `mountables/token.txt` — файл с Telegram Bot Token; его нужно создать вручную перед запуском.

## Как устроен пайплайн

Голосование создаётся и запускается в несколько последовательных шагов.

### Шаг 1. Подготовьте исходные фотографии

Создайте папку с категориями. Каждая подпапка — это одна категория голосования, а внутри лежат фотографии этой категории.

Пример исходной структуры:

```text
entries/
├── Категория 1/
│   ├── img_001.jpg
│   ├── img_002.jpg
│   └── img_003.jpg
├── Категория 2/
│   ├── photo1.jpg
│   ├── photo2.jpg
│   └── photo3.jpg
└── Категория 3/
    ├── DSC001.jpg
    ├── DSC002.jpg
    └── DSC003.jpg
```

На этом этапе названия категорий и файлов могут быть произвольными, но лучше:
- избегать совсем экзотических символов;
- не использовать вложенные подпапки;
- хранить внутри только изображения;
- заранее удалить мусорные файлы

---

### Шаг 2. Приведите структуру к формату бота

Скрипт `rename_all_entries.py` преобразует имена папок и файлов в предсказуемую структуру, которую использует бот.

После выполнения структура должна выглядеть примерно так:

```text
mountables/
└── entries/
    ├── PACK_1/
    │   ├── 1.jpg
    │   ├── 2.jpg
    │   └── 3.jpg
    ├── PACK_2/
    │   ├── 1.jpg
    │   ├── 2.jpg
    │   └── 3.jpg
    └── PACK_3/
        ├── 1.jpg
        ├── 2.jpg
        └── 3.jpg
```

### Как запускать

Положите исходные категории в папку entries/ и выполните:

```bash
python rename_all_entries.py
```

Проверьте вручную:
- что все категории стали папками вида `PACK_1`, `PACK_2`, `PACK_3`, ...;
- что внутри каждого `PACK_N` файлы имеют последовательные имена;
- что все изображения действительно открываются;
- что нет пропусков в нумерации.

## Создание таблицы голосования

После подготовки файлов нужно создать таблицу для нового года в SQLite-базе.

### Что делает `creating_voting_db.py`

Скрипт:
- подключается к `mountables/voting.db`;
- определяет число категорий;
- создаёт новую таблицу голосования;
- не затрагивает старые таблицы прошлых лет.

Обычно логика такая:
- одна таблица = один год голосования;
- пример имени таблицы: `voting_2026`;
- столбцы выглядят как `id`, `PACK_1`, `PACK_2`, `PACK_3` и т. д.

### Запуск

```bash
python creating_voting_db.py
```

### Что важно проверить

После запуска желательно убедиться, что:
- база `mountables/voting.db` существует;
- новая таблица действительно создана;
- число столбцов `PACK_N` совпадает с числом папок `PACK_N` в `mountables/entries`.

Это можно проверить через любой SQLite viewer или через Python.

Пример проверки в Python:

```python
import sqlite3

conn = sqlite3.connect("mountables/voting.db")
cursor = conn.cursor()

cursor.execute("SELECT name FROM sqlite_master WHERE type='table' ORDER BY name;")
print(cursor.fetchall())
```

### 1. Создайте виртуальное окружение

```bash
pip install -r requirements.txt
```

### 3. Создайте файл с токеном

Создайте файл:

И поместите туда токен Telegram-бота:

- `mountables/token.txt`

### 5. Запустите бота

```bash
python main.py
```

Если всё корректно настроено, бот перейдёт в режим polling и начнёт принимать команды в Telegram.

## Базовый сценарий использования

Пользовательский сценарий обычно такой:

1. Пользователь пишет `/start`.
2. Бот отправляет приветствие и инструкцию.
3. Пользователь пишет `/vote`.
4. Бот отправляет фотографии из первого пака.
5. Пользователь нажимает кнопку с номером выбранной фотографии.
6. Бот сохраняет голос в БД и переходит к следующему паку.
7. После последнего пака бот сообщает, что голосование завершено.
8. Если нужно переголосовать, пользователь использует `/cancel`, затем снова `/vote`.

### Что хранится в БД

Обычно в таблице одна строка соответствует одному Telegram user id.

Пример:

| id | PACK_1 | PACK_2 | PACK_3 |
|---|---:|---:|---:|
| 111111111 | 3 | 1 | 5 |
| 222222222 | 2 | 4 | 1 |

Это означает:
- пользователь `111111111` выбрал фото 3 в первой категории;
- фото 1 во второй;
- фото 5 в третьей.

## Как извлекать результаты

После окончания голосования бот можно остановить, а результаты прочитать напрямую из SQLite.

### Вариант 1. Посмотреть все голоса

```python
import sqlite3

conn = sqlite3.connect("mountables/voting.db")
cursor = conn.cursor()

cursor.execute("SELECT * FROM voting_2026")
rows = cursor.fetchall()

for row in rows:
    print(row)
```

### Вариант 2. Посчитать победителя по каждому паку

Пример для одного пака:

```python
import sqlite3
from collections import Counter

conn = sqlite3.connect("mountables/voting.db")
cursor = conn.cursor()

cursor.execute("SELECT PACK_1 FROM voting_2026 WHERE PACK_1 IS NOT NULL")
votes = [row for row in cursor.fetchall()]

counter = Counter(votes)
print(counter.most_common())
```

## Запуск на сервере

Для длительного голосования лучше запускать бота не на личном компьютере, а на сервере.

### Подготовка сервера

Подключитесь по SSH:

```bash
ssh user@your_server_ip
```

Установите Python и системные утилиты:

```bash
sudo apt update
sudo apt install -y python3 python3-venv python3-pip git
```

Скопируйте проект на сервер:
- через `git clone`, если проект в репозитории;
- или через `scp` / WinSCP / FileZilla, если проект локальный.

Пример через `scp`:

```bash
scp -r ./your_project user@your_server_ip:/home/user/photo_voting_bot
```

### Настройка проекта на сервере

```bash
cd /home/user/photo_voting_bot
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Создайте файл токена:

```bash
mkdir -p mountables
nano mountables/token.txt
```

Вставьте токен, сохраните файл.

После этого:
1. загрузите фотографии;
2. выполните `rename_all_entries.py`;
3. выполните `creating_voting_db.py`;
4. выполните `main.py`.

## Запуск через Docker

Docker полезен, если нужны:
- воспроизводимое окружение;
- простой деплой на сервер;
- изоляцию зависимостей;
- удобное обновление контейнера.

### Пример `Dockerfile`

```dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["python", "main.py"]
```

### Важное замечание про данные

Папка `mountables` должна храниться вне контейнера или быть примонтирована как volume, иначе:
- токен может потеряться при пересоздании контейнера;
- база `voting.db` тоже потеряется;
- логи не сохранятся между перезапусками.

### Сборка образа

```bash
docker build -t photo-voting-bot .
```

### Запуск контейнера

```bash
docker run -d \
  --name photo-voting-bot \
  --restart unless-stopped \
  -v $(pwd)/mountables:/app/mountables \
  photo-voting-bot
```

### Если сервер Windows

На Windows Docker-команда будет другой по синтаксису путей, но идея та же:
- код находится в контейнере;
- `mountables` монтируется с хоста;
- БД и токен хранятся вне контейнера.

## Рекомендуемый рабочий процесс на каждый новый год

Ниже практический порядок действий, который удобно повторять каждый сезон.

### 1. Подготовьте новый набор фотографий

Соберите все категории и изображения для нового голосования.

### 2. Сделайте резервную копию базы

Перед созданием новой таблицы всегда делайте backup:

```bash
cp mountables/voting.db mountables/voting_backup_2026.db
```

### 3. Подготовьте папки `entries`

Разложите фотографии по категориям.

### 4. Запустите переименование

```bash
python rename_all_entries.py
```

### 5. Создайте таблицу нового года

```bash
python creating_voting_db.py
```

### 6. Проверьте, что имя таблицы в коде соответствует текущему году

В коде имя таблицы строится автоматически, просто убедитесь, что системная дата сервера правильная.

### 7. Запустите бота

Локально:
```bash
python main.py
```

На сервере:
```bash
sudo systemctl start photo-voting-bot
```

### 8. Проведите тест перед реальным запуском

Обязательно протестируйте:
- `/start`
- `/vote`
- прохождение нескольких паков
- `/cancel`
- повторное голосование
- поведение после перезапуска бота

## Что делать при перезапуске бота

Если бот был остановлен и запущен снова:
- база SQLite останется на месте;
- уже записанные голоса не потеряются;
- пользователи смогут продолжить голосование.

Именно поэтому БД и папку `mountables` важно хранить на диске сервера, а не во временном каталоге.

## Частые проблемы

### Бот не запускается
Проверьте:
- существует ли `mountables/token.txt`;
- правильный ли токен внутри;
- установлены ли зависимости;
- существует ли `mountables/voting.db`;
- создана ли таблица нужного года.

### Бот пишет, что таблица не найдена
Скорее всего:
- не был выполнен `creating_voting_db.py`;
- имя таблицы в `main.py` не совпадает с созданной таблицей;
- вы подключились не к той базе.

### Фото не отправляются
Проверьте:
- существуют ли папки `PACK_N`;
- есть ли внутри реальные изображения;
- совпадают ли расширения файлов и содержимое;
- не сломались ли имена файлов после переименования.

### Пользователь уже голосовал, но хочет начать заново
Нужно использовать:

```text
/cancel
```

а затем:

```text
/vote
```
- `photo-voting-bot.service` для `systemd`
- маленький `export_results.py` для выгрузки итогов в CSV

Отдельно полезно добавить раздел “Release checklist”, чтобы перед каждым новым сезоном голосования вы не вспоминали процесс вручную, а шли по фиксированному списку шагов.

Следующим сообщением я могу сразу подготовить вам **готовые файлы** `Dockerfile`, `docker-compose.yml`, `.gitignore` и `photo-voting-bot.service` в копируемом виде.
